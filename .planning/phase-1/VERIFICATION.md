### 问题

**FLAG-2: Plan 01-01 包含 6 个任务 + 2 个前置条件，影响文件数约 15+，超出建议范围（2-3 任务/Plan 为目标，4 为警告阈值）。**

Plan 01-01 覆盖了: Wails CLI 安装、项目初始化、配置文件、Go 后端目录结构、前端目录结构、Vue Router 配置、构建验证。这实际上是 Wails 脚手架生成 + 目录结构规划 + 基础配置三件事的合并。

**改进建议**: 考虑将 Plan 01-01 拆分为:
- 01-01a: Wails 初始化 + 基础构建配置（wails init + wails.json + main.go + app.go + 验证构建）
- 01-01b: 目录结构规划 + 基础路由配置（Go internal 结构 + 前端结构 + Vue Router）

不过，鉴于这些任务高度关联（Wails 脚手架会一次性生成大部分文件），拆分可能导致执行时的衔接问题。此为 FLAG 而非 BLOCK，执行中注意上下文预算即可。

**FLAG-3: 计划文件缺少 GSD 标准任务格式。**

gsd-tools 验证结果:
```
Missing required frontmatter fields: phase, plan, type, wave, depends_on, files_modified, autonomous, must_haves
No <task> elements found
```

PLAN.md 使用自由格式描述而非 GSD 标准模板格式（`<task>` XML 元素 + YAML frontmatter）。这导致:
1. gsd-tools 无法解析和验证计划结构
2. 缺少 `must_haves`（truths/artifacts/key_links）导致关键工件和关联无法追踪
3. 缺少 `<verify>` 元素导致自动化验证命令缺失
4. 缺少 `wave` / `depends_on` 元素导致依赖图无法自动构建

**改进建议**: 将 PLAN.md 转换为 GSD 标准格式，每个 Plan 一个独立的 `*-PLAN.md` 文件（如 `01-01-PLAN.md`, `01-02-PLAN.md`, `01-03-PLAN.md`），包含完整的 YAML frontmatter 和 `<task>` XML 元素。这会显著提升自动化工具的可用性。
