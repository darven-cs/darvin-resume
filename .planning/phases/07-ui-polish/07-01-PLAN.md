# 07-01: 设计令牌与主题系统

**Plan**: 07-01
**Wave**: 1（基础设施）
**覆盖需求**: UIUX-01, UIUX-02, UIUX-03
**预计新增/修改文件**: 12个

## 目标

建立统一的UI设计令牌体系和深色/浅色主题切换系统，使所有UI组件使用CSS变量驱动颜色、间距、字体，支持三模式切换（浅色/深色/跟随系统）。

### Task 2: 创建主题切换 composable `frontend/src/composables/useTheme.ts`

```typescript
type ThemeMode = 'light' | 'dark' | 'system'

// 核心功能:
// - 读取/写入 localStorage + 后端settings
// - 监听 prefers-color-scheme 变化
// - 设置 document.documentElement.dataset.theme
// - 同步 Monaco 编辑器主题 (vs / vs-dark)
// - 提供 isDark computed
// - 通过 Wails 事件通知后端窗口背景色变更
```

关键逻辑:
- `system` 模式下监听 `window.matchMedia('(prefers-color-scheme: dark)')` 的 `change` 事件
- 主题切换时发射 Wails 事件 `theme-changed`，后端监听更新窗口背景色
- Monaco 编辑器通过 `monaco.editor.setTheme()` 同步切换

### Task 3: 后端主题设置 `app.go` 新增 Bridge 方法

```go
// GetTheme() → string ("light" | "dark" | "system")
// SetTheme(theme string) → error
```

使用现有 `internal/settings` 基础设施:
- Key: `ui.theme`
- 默认值: `"system"`
- SetTheme 校验值必须是三个有效值之一

### Task 4: 更新后端窗口背景色同步

`app.go` startup 中初始化主题:
- 读取 `ui.theme` settings
- 盘听前端 `theme-changed` 事件
- 调用 Wails runtime 更新窗口背景色:
  - 浅色: `RGBA{R: 255, G: 255, B: 255, A: 1}`
  - 深色: `RGBA{R: 30, G: 30, B: 30, A: 1}`

### Task 5: 升级 AIConfigModal 为通用设置对话框 `SettingsDialog.vue`

将现有 `AIConfigModal.vue`（仅含AI配置）升级为多Tab通用设置对话框:

**Tab 1: AI 配置** — 迁移现有 AIConfigModal 内容（API Key / Base URL / 模型 / Token / 超时）

**Tab 2: 外观** — 新增主题设置
- 主题模式选择: 浅色 / 深色 / 跟随系统（radio按钮组）
- 当前模式状态文字提示
- 切换即时生效，无需保存按钮

**Tab 3: 快捷键** — 由 07-03 Plan 添加

**UI 结构**:
```
┌─ 设置 ──────────────────────────┐
│ [AI 配置]  [外观]  [快捷键]      │ ← Tab 栏
│─────────────────────────────────│
│                                 │
│  (当前Tab内容)                   │
│                                 │
│              [关闭]              │
└─────────────────────────────────┘
```

**设置入口**:
- HomeView 顶部齿轮按钮 → 打开 SettingsDialog
- EditorView 工具栏 → **删除** 现有"设置"按钮（设置集中到首页入口）
- Sidebar 底部 → 可选添加齿轮图标入口

同时删除 `EditorView.vue` 中的"设置"按钮和 `showAIConfigModal` 状态。

### Task 6: 迁移 App.vue 全局样式到设计令牌

将 `App.vue` 中的硬编码颜色替换为 CSS 变量:
- `background-color: #1e1e1e` → `var(--ui-bg-primary)`
- `color: #e0e0e0` → `var(--ui-text-primary)`
- sidebar `background-color: #252526` → `var(--ui-bg-sidebar)`
- 所有组件中重复的颜色值逐一替换

**替换策略** (按优先级):
1. `App.vue` 全局样式
2. `HomeView.vue` 主页
3. `EditorView.vue` 工具栏和布局
4. Sidebar 组件
5. 其他组件（BackupManager, ExportDialog, SettingsDialog等）

### Task 7: 迁移各组件硬编码颜色

逐个组件扫描替换，统一使用 `var(--ui-*)` 变量:
- `HomeView.vue` — top-bar背景、文字、卡片样式
- `EditorView.vue` — 工具栏、状态栏
- `AIChatSidebar.vue` — 侧边栏背景、消息气泡
- `MonacoEditor.vue` — 容器背景（Monaco自身主题由Task 2控制）
- `TemplateSelector.vue` — 模板卡片样式
- `StyleEditor.vue` — 样式编辑器
- `ExportDialog.vue` — 对话框样式
- `BackupManager.vue` — 备份管理界面
- `SnapshotManager.vue` — 快照管理界面
- `ContextMenu.vue` — 右键菜单
- `SaveStatusIndicator.vue` — 状态指示器

## 验收标准

2. ✅ 三模式（浅色/深色/跟随系统）均正确工作
3. ✅ Monaco 编辑器主题随 UI 主题同步切换
4. ✅ 简历预览区域不受主题切换影响（始终保持白色A4纸张+模板样式）
5. ✅ 主题选择持久化，重启应用后保持上次选择
6. ✅ 所有组件无硬编码颜色值，全部使用 `var(--ui-*)` 变量
7. ✅ 设置对话框包含"AI 配置"和"外观"两个Tab，快捷键Tab由07-03添加
8. ✅ EditorView 工具栏的"设置"按钮已删除
9. ✅ `go build ./...` 和前端编译均通过
