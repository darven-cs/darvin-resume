# Plan 04-02: AI 引导式创建流程 — 侧边栏向导

**Phase:** 4 of 7 — 简历创建与管理
**Status:** Planned
**Depends on:** 04-01 (CreateModeModal, HomeView)

## Goal

实现 AI 引导式简历创建向导，用户在编辑器右侧侧边栏中分步完成模块勾选、内容填写、AI 润色，最终生成完整 Markdown 简历。

## Requirements Covered
- **RESM-01**: AI 引导模式 — 分步填写 + AI 润色
- **RESM-02**: 模块勾选 — 用户选择要包含的简历模块
- **RESM-03**: 分步填写 — 按勾选顺序逐模块填写表单
- **RESM-04**: AI 润色 — 每个模块手动触发 AI 润色，对比展示

## Deliverables

- 底部操作按钮区

### 2. 步骤一：模块勾选 (`WizardModuleSelect`)

**功能 per D-06:**
- 展示可选模块列表，每项一个勾选框：
  - 默认勾选：基础信息、专业技能、项目经历、自我评价
  - 可选勾选：校园经历、实习经历、获奖、证书
- 每个模块显示图标 + 名称 + 简短描述
- 底部「开始填写」按钮（至少勾选 1 个模块才可点击）

**模块定义：**
```typescript
interface WizardModule {
  type: string       // 'basicInfo' | 'skills' | 'projects' | 'evaluation' | 'campus' | 'internship' | 'awards' | 'certificates'
  label: string      // 显示名称
  icon: string       // 图标
  description: string // 简短描述
  defaultChecked: boolean
}
```

### 3. 步骤二：逐步填写 (`WizardStepForm`)

**功能 per D-07~D-09:**
- 按 Phase 1 勾选顺序，逐个模块展示填写表单
- 顶部显示当前模块名称 + 进度（如 "2/5 专业技能"）
- 每个模块的表单字段：

| 模块 | 字段 |
|------|------|
| 基础信息 | 姓名、电话、邮箱、GitHub、个人网站、地址、一句话简介 |
| 专业技能 | 技能类别 + 技能列表（可动态添加） |
| 项目经历 | 项目名、角色、时间、描述、技术栈、亮点（可多项） |
| 自我评价 | 评价内容（textarea） |
| 校园经历 | 活动名、角色、时间、描述、亮点 |
| 实习经历 | 公司、职位、时间、描述、亮点 |
| 获奖 | 奖项名、级别、日期、描述 |
| 证书 | 证书名、颁发机构、日期、成绩 |

- 每个表单字段：标签 + 输入框 + placeholder 示例提示
- 底部按钮：「AI 润色」+ 「跳过」+ 「下一步」
- AI 润色：手动触发 per D-08，润色结果在表单下方以对比方式展示
- 跳过：保留空模板 per D-09，直接进入下一模块
- 支持重新润色、手动修改润色结果 per D-09

### 4. 步骤三：生成简历 (`WizardGenerate`)

**功能 per D-10:**
- 展示所有已填写模块的摘要预览
- 底部「生成简历」按钮
- 点击后：
  1. 将所有模块数据组合为 Resume JSON 结构
  2. 调用 `UpdateResume(id, jsonData)` 保存到后端
  3. 后端自动执行 JSON → Markdown 转换
  4. 触发 `@complete` 事件，向导关闭，编辑器显示生成内容

### 5. 中途退出处理 per D-11

- 用户点击关闭按钮时弹出确认框：「退出将保存为草稿，是否继续？」
- 确认：已填写内容生成部分 Markdown → 调用 `UpdateResume` 保存 → 关闭向导
- 取消：继续填写

### 6. EditorView 集成

**修改 `EditorView.vue`：**
- 检测路由 query `?wizard=true`，自动打开 `ResumeWizardSidebar`
- 向导关闭后切换到正常编辑模式
- 向导完成后通过 `UpdateResume` 保存内容并刷新编辑器

```typescript
// EditorView onMounted 中
const showWizard = computed(() => route.query.wizard === 'true')
const wizardVisible = ref(showWizard.value)

function handleWizardComplete() {
  wizardVisible.value = false
  // 刷新编辑器内容
  loadResume()
}

function handleWizardSaveDraft() {
  wizardVisible.value = false
  loadResume()
}
```

### 7. AI 润色实现

**调用方式：**
- 复用现有 `AISendMessageSync` 非流式调用（润色需要完整结果才能展示对比）
- 构建 prompt：将当前模块的原始填写内容作为 user message
- System prompt：`你是专业的简历优化助手。请优化以下{模块名}内容，使其更加专业和有吸引力。只返回优化后的内容，不要添加解释。`

**对比展示：**
- 原文和润色结果左右/上下对比显示
- 「接受」按钮：用润色结果替换表单内容
- 「重新润色」按钮：再次调用 AI
- 「保持原样」按钮：丢弃润色结果

## Technical Details

### 模块表单数据结构

```typescript
// 向导状态管理
interface WizardState {
  step: 'select' | 'fill' | 'generate'
  selectedModules: string[]    // 勾选的模块列表
  currentModuleIndex: number   // 当前填写的模块索引
  formData: Record<string, any> // 各模块填写数据
  polishResults: Record<string, { original: string, polished: string }> // 润色结果
}
```

### 模块→JSON→Markdown 转换

生成简历时，将 formData 组装为标准 Resume JSON 结构（匹配 `internal/model/resume.go` 中的 Resume struct），调用 `UpdateResume` 后端自动执行 `ConvertResumeToMarkdown` 转换。
