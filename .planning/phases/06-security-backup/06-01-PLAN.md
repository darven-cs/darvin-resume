创建 `internal/crypto/` 目录和 `aes.go` 文件。

实现 AES-256-GCM 认证加密：

1. **常量定义：**
   - KeyVersion = 1（预留版本兼容）
   - KeyLength = 32（AES-256）
   - SaltLength = 16
   - NonceLength = 12（GCM 标准）
   - Argon2Time = 1, Argon2Memory = 64*1024 (64MB), Argon2Threads = 4

2. **Encrypt(plaintext []byte, passphrase string) ([]byte, error)：**
   - 生成 16 字节随机 salt（crypto/rand）
   - Argon2id 派生 32 字节密钥
   - 创建 AES-GCM cipher
   - 生成 12 字节随机 nonce
   - Seal 加密（自动追加 16 字节 auth tag）
   - 组装输出：version(1B) + salt(16B) + nonce(12B) + ciphertext+tag
   - 清除内存中的密钥

3. **Decrypt(encrypted []byte, passphrase string) ([]byte, error)：**
   - 验证密文最小长度
   - 解析 version/salt/nonce/ciphertext
   - Argon2id 派生同一密钥
   - Open 解密 + 验证认证标签
   - 清除内存中的密钥

4. **注意事项：**
   - 使用 `golang.org/x/crypto/argon2`（已在 go.mod 间接依赖）
   - 包名为 `crypto`（`internal/crypto` 是合法路径，不与标准库 `crypto` 冲突）
   - 如果需要区分，可以用 `internal/encryption` 作为包名避免歧义

  <action>
创建 `internal/crypto/device.go`，实现跨平台的设备唯一标识获取。

**DeviceKey() (string, error) 函数：**

1. **Windows**（`runtime.GOOS == "windows"`）：
   - 执行 `powershell -Command "Get-ItemProperty 'HKLM:\SOFTWARE\Microsoft\Cryptography' | Select-Object -ExpandProperty MachineGuid"`
   - 提取 MachineGuid 字符串

2. **macOS**（`runtime.GOOS == "darwin"`）：
   - 执行 `ioreg -rd1 -c IOPlatformExpertDevice`
   - 从输出中提取包含 `IOPlatformUUID` 的行的 UUID 值

3. **Linux**（`runtime.GOOS == "linux"`）：
   - 读取 `/etc/machine-id`
   - 如果失败，fallback 读取 `/var/lib/dbus/machine-id`

4. **最终处理：**
   - 如果 seed 为空，返回错误
   - 拼接应用标识：`"Darvin-Resume-DeviceKey-V1:" + seed`
   - SHA-256 哈希，返回 hex 编码字符串
   - 这样同一设备不同应用得到不同密钥

创建 `internal/ai/secure_config.go`，实现 API Key 的加密存储和安全读取。

**SaveSecureAPIKey(ctx context.Context, apiKey string) error：**
1. 如果 apiKey 为空，直接存储空字符串到 settings
2. 调用 `crypto.DeviceKey()` 获取设备密钥
3. 调用 `crypto.Encrypt([]byte(apiKey), deviceKey)` 加密
4. 存储为 `"enc:" + hex.EncodeToString(encrypted)` 到 settings

**LoadSecureAPIKey(ctx context.Context) (string, error)：**
1. 从 settings 读取 `ai.apiKey` 的值
2. 如果为空，返回空字符串
3. 如果不以 `enc:` 开头（明文旧值）：
   - 触发自动迁移：调用 SaveSecureAPIKey 重新加密存储
   - 返回明文值（向后兼容）
4. 如果以 `enc:` 开头：
   - 去掉前缀，hex 解码
   - 获取设备密钥
   - 调用 crypto.Decrypt 解密
   - 返回明文 API Key

**注意事项：**
- 使用 `encoding/hex` 编码密文
- 迁移过程是透明的，不需要用户干预
- `strings.HasPrefix` 检测 `enc:` 前缀
- 包名为 `ai`（与现有文件同包）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 设备密钥生成 | 依赖平台 API（Windows 注册表/macOS ioreg/Linux machine-id），如果设备特征被篡改则无法解密 |
| Argon2id 参数 | 64MB/4线程是安全与性能的平衡点，桌面应用可接受 |
| 密钥内存清零 | Go GC 可能将密钥拷贝到其他位置，但这是 Go 语言固有限制 |

## Mitigations
- 设备密钥不存储文件，每次动态计算
- 每次加密使用随机 salt + nonce，防重放
- GCM 认证标签确保数据完整性
</threat_model>
