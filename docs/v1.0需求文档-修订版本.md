## 1. 文档信息
- 项目名称：Darvin-Resume
- 目标平台：Windows、macOS、Linux
- 文档范围：本文件为Darvin-Resume v1.0 MVP版本的完整产品需求定义，是开发、测试、迭代的唯一权威依据
- 目标受众：开发团队、测试团队、产品迭代负责人
- 核心定位：一款基于Markdown原生、隐私优先、AI深度协同、开发者友好的本地化简历制作与管理工具

## 2. 项目概述
### 2.1 目标用户定义
**MVP阶段核心目标用户**：计算机相关专业应届生/毕业3年内的初级开发者，聚焦校招、社招初级岗位的简历制作需求；资深开发者、跨行业转行等场景放入v1.1及后续迭代。

### 2.2 产品目标与量化验收标准
#### 核心产品目标
1.  解决开发者简历制作核心痛点：排版错乱、多版本管理难、内容润色效率低、简历隐私泄露风险
2.  以本地部署为核心，确保用户简历数据100%留存本地，无强制云端上传风险
3.  以AI能力为提效核心，降低优质简历的制作门槛
4.  提供贴合开发者习惯的、接近VS Code的专业交互体验

#### 强制量化验收标准（v1.0必须达成）
1.  编辑器体验：Monaco Editor 100%兼容VS Code默认编辑键位，支持Markdown语法高亮与基础补全
2.  性能指标：软件冷启动速度<2s，常态运行内存占用<200MB，单页简历PDF导出耗时<3s
3.  体验保障：编辑器预览与PDF导出排版100%一致，无错位、断行、样式偏差
4.  稳定性指标：核心功能场景崩溃率<0.1%，无任何用户数据丢失场景

### 2.3 核心价值
1.  **隐私优先**：全量数据本地存储于SQLite数据库，无强制云端上传；用户自备AI API Key，平台不截留任何请求数据
2.  **AI深度协同**：内置开发者简历专属AI智能体，MVP阶段聚焦**内容润色、结构化生成、翻译/缩写、格式对齐**四大核心能力，精准贴合开发者简历制作场景
3.  **所见即所得**：Markdown编辑、实时预览、PDF导出采用统一渲染引擎，确保编辑效果与最终导出效果100%一致
4.  **开发者友好**：原生支持Markdown语法，兼容VS Code编辑习惯，轻量化安装包，低资源占用

### 2.4 MVP边界与迭代规划
#### v1.0 MVP 必须交付核心功能
- 核心Markdown编辑器（Monaco内核）与实时预览功能
- AI引导简历生成、选中文本AI润色/翻译/缩写核心能力
- 内置简历模板与基础样式自定义能力
- 本地数据存储、简历版本管理、PDF导出核心功能
- AI API接口适配与安全配置能力

#### v1.1 迭代规划（v1.0交付后启动）
- ATS简历优化、岗位匹配度分析功能
- AI智能内容补全、简历内容查重能力
- PDF/Word简历OCR解析功能
- 资深开发者专属简历模板与引导逻辑

#### 永久不纳入产品范围的功能
- 云端简历存储、共享、投递功能（坚守本地化隐私定位）
- 招聘平台对接、内推信息整合功能
- 企业端简历筛选、人才管理相关功能

## 3. 功能需求
### 3.1 核心编辑器模块
本模块为产品核心交互载体，基于Monaco Editor（VS Code内核）构建，为用户提供贴合开发者习惯的Markdown简历编辑能力。

#### 3.1.1 简历创建功能
1.  **AI引导式创建**
    - 前置模块勾选环节：用户创建时可自定义勾选需要填写的简历模块，默认勾选「基础信息、专业技能、项目经历、自我评价」，可选模块包含「校园经历、实习经历、获奖经历、证书资质」，非勾选模块无需进入填写步骤，直接跳过
    - 引导流程：用户勾选模块后，按自定义模块顺序进入分步填写页，每个步骤提供明确的填写框架与示例（如项目经历需按「项目背景、个人职责、技术栈、量化成果」框架填写）
    - 生成逻辑：用户每完成一个步骤的填写，AI实时润色内容并生成符合规范的结构化数据，全部步骤完成后自动生成完整Markdown简历与对应预览效果
    - 操作权限：支持单步骤内容重生成、单步骤跳过、流程中途退出并保存已填写内容
2.  **空白页创建**
    - 用户选择后直接进入双栏编辑界面，生成空白Markdown简历模板，用户可自行编辑，全程可随时调用AI能力进行内容生成、润色、调整

#### 3.1.2 Markdown编辑核心能力
1.  编辑器内核：采用Monaco Editor，100%兼容VS Code默认编辑键位
2.  语法高亮：完整支持标准Markdown语法高亮，适配简历常用的标题、列表、表格、粗体、斜体、分割线等语法
3.  自动补全：MVP阶段仅支持Markdown标准语法自动补全，AI智能内容补全放入后续迭代
4.  基础编辑能力：支持撤销/重做、多光标编辑、块选择、查找替换等VS Code标配编辑功能

#### 3.1.3 实时预览功能
1.  布局规则：默认双栏布局，左侧为Markdown编辑区，右侧为实时预览区，支持拖拽调整双栏宽度比例
2.  **渲染一致性强制规则**：编辑器预览区与PDF导出功能采用完全一致的渲染引擎与CSS样式表，确保100%所见即所得，无样式、排版、分页偏差
3.  分页提示：预览区固定显示A4标准纸张分页线，明确标注页面边界，用户编辑时可实时感知内容分页情况，避免导出后排版错乱
4.  同步规则：编辑区内容输入后，预览区实时同步渲染，延迟<200ms

#### 3.1.4 编辑交互增强
1.  选区快捷操作：用户框选编辑区内文本后，自动弹出浮动工具栏，包含「AI润色、翻译、缩写、重写」核心快捷动作，点击即可触发对应AI能力
2.  行级交互能力：
    - 行首符号：Markdown标题、列表块行首显示折叠/展开图标，支持点击折叠对应内容块
    - 拖拽排序：支持长按行首图标拖拽，快速调整内容块的上下顺序
    - 快捷菜单：点击行首图标弹出快捷菜单，包含「上移、下移、AI重写、删除」四个核心操作
3.  Diff对比能力：AI修改内容后，以Diff视图（删除内容标红、新增内容标绿）展示修改前后差异，用户点击「接受」才将修改应用到编辑区，点击「拒绝」保留原内容

### 3.2 AI智能体模块
本模块为产品核心提效能力，基于Claude Messages API构建，聚焦开发者简历制作场景，提供稳定、精准、可控的AI辅助能力，所有能力均严格遵循MVP边界定义。

#### 3.2.1 接口适配能力
1.  核心适配：首选集成Claude Messages API，为MVP阶段官方支持的唯一接口
2.  扩展适配：支持用户自定义BaseURL，以兼容API中转站服务，预留接口适配层，后续新增模型仅需新增适配器，无需修改核心逻辑
3.  **安全存储规范**：用户输入的API Key采用AES-256-GCM算法加密存储，密钥基于设备唯一标识生成，仅当前设备可解密，杜绝明文存储泄露风险
4.  配置管理：支持用户配置默认模型、单次调用最大Token数、请求超时时间，配置项本地存储

#### 3.2.2 数据规范与上下文管理
1.  **结构化数据强制规范**：
    - 所有AI生成、修改操作，均以附件中**简历结构化JSON Schema**为唯一标准，AI输出必须严格符合Schema规范，禁止输出无约束的自由文本
    - 前后端、AI生成、模板渲染全流程共用同一套JSON Schema规范，确保数据格式完全统一
2.  **职位目标上下文管理**：
    - 在简历结构化JSON中新增固定字段`target_position`，用于存储用户当前简历的目标岗位
    - 入口规则：用户可在简历编辑页顶部固定区域编辑目标岗位，编辑后实时同步到结构化JSON中
    - 调用规则：每次AI能力调用，默认传入该字段内容，确保AI生成内容贴合目标岗位需求；同时支持用户在单次调用弹窗中临时修改目标岗位，仅对本次调用生效
3.  全文上下文管理：
    - 每次AI调用，提供「全文参考」开关，用户开启后，请求将附带简历核心结构化信息（目标岗位、核心技能、工作年限），确保AI生成内容与整篇简历的语气、风格、内容保持一致，避免前后矛盾
    - 开关默认规则：局部润色/翻译/缩写默认关闭全文参考，全文重写/结构化生成默认开启全文参考
4.  流式传输：所有AI生成内容均支持流式传输，实现打字机效果实时渲染，提升交互丝滑感，无长时间空白等待

#### 3.2.3 交互模式
1.  引导式填写交互：对应3.1.1中的AI引导创建流程，按模块分步引导用户输入信息，AI实时润色并生成结构化数据
2.  选区快捷交互：对应3.1.4中的选区快捷操作，用户选中文本后一键触发对应AI能力，结果以Diff视图展示
3.  侧边栏对话交互：编辑器右侧支持唤起AI对话侧边栏，支持多轮对话，可引用编辑区内选中文本作为上下文，支持用户自由提问，AI输出内容可一键插入到编辑区
4.  一键解析功能：
    - **MVP边界定义**：仅支持纯文本/Markdown格式内容的结构化解析，不支持PDF/Word文件解析与OCR能力，相关功能放入v2.0迭代
    - 功能逻辑：用户粘贴旧简历文本内容后，AI自动解析并生成符合JSON Schema规范的结构化数据，同步生成对应Markdown内容，适配当前模板

#### 3.2.4 异常场景兜底策略（强制执行）
所有AI调用场景必须包含以下兜底策略，杜绝用户内容丢失、体验崩盘：
1.  网络/API调用失败：弹窗明确提示错误原因，完整保留用户原始输入内容，提供「一键重试」按钮，重试不消耗用户额外输入成本
2.  Token超限场景：自动检测用户输入内容长度，超出模型Token限制时，自动拆分内容分批生成，同时弹窗提示用户精简选中文本，避免生成内容截断
3.  AI返回格式异常：后端接收到AI返回内容后，先基于JSON Schema进行格式校验，校验不通过自动重试1次；重试仍失败则完整返回用户原始内容，弹窗提示格式校验失败，允许用户手动编辑，绝不丢失用户输入内容
4.  内容生成中断：用户主动中断生成时，保留已生成的有效内容，允许用户继续编辑或重新生成

### 3.3 模板与渲染系统
本模块负责简历内容的样式渲染与格式管理，核心目标是实现样式与数据分离，确保渲染一致性，同时提供灵活的自定义能力。

#### 3.3.1 模板管理能力
1.  内置模板：MVP阶段默认内置4套标准化模板，分别为「极简通用风、大厂校招风、学术科研风、双栏简约风」，所有模板均严格遵循统一的JSON Schema规范
2.  **模板切换强制规则**：切换模板仅修改简历的渲染样式，底层结构化JSON数据完全复用，确保切换模板后内容无丢失、无错乱
3.  模板管理：支持用户将自定义样式保存为个人模板，可复用至新建简历中

#### 3.3.2 样式自定义能力
1.  可视化样式调整：支持用户通过滑块/选择器调整核心样式参数，包括主色调、基础字号、行高、页面边距、字体选择，无需编写CSS代码，调整后实时预览效果
2.  自定义CSS能力：
    - **安全白名单机制**：MVP阶段仅开放字体、颜色、边距、行高、列表样式相关的CSS属性白名单，禁止修改布局、分页、定位相关的核心属性，避免渲染崩溃、PDF导出异常
    - 兜底方案：提供「一键重置默认样式」功能，用户自定义CSS出现异常时，可一键恢复到模板默认样式
    - 生效范围：自定义CSS仅对当前简历生效，不影响其他简历与模板

#### 3.3.3 JSON-MD双向同步规则（强制执行）
**核心原则**：结构化JSON为全系统唯一可信数据源，Markdown编辑器为视图展示层，所有核心数据操作均基于JSON进行，避免数据错乱。
1.  **正向同步（JSON → Markdown）**：
    - 结构化JSON数据发生修改（含AI生成、修改操作）后，系统自动基于当前模板生成对应Markdown内容，实时同步到编辑器中
    - 正向同步为自动执行，无需用户手动触发，确保数据一致性
2.  **反向同步（Markdown → JSON）**：
    - 用户在Markdown编辑器中手动修改内容后，需主动点击「同步到结构化数据」按钮，触发反向同步
    - 同步逻辑：由AI解析用户修改后的Markdown内容，更新对应结构化JSON字段，更新前以Diff视图展示修改差异，用户确认后再执行更新
    - 禁止自动反向同步：避免自动解析Markdown导致的格式错乱、数据丢失，确保JSON数据源的稳定性

### 3.4 数据管理与导出模块
本模块负责用户简历数据的全生命周期管理，包括本地存储、版本管理、PDF导出、备份恢复，核心目标是保障数据安全、可追溯、导出效果可控。

#### 3.4.1 PDF导出功能
1.  **核心导出方案**：MVP阶段默认采用Wails原生系统打印接口实现PDF导出，无需内置Chromium，大幅压缩安装包体积，降低内存占用，确保与轻量化选型优势匹配
2.  高级渲染备选方案：保留Chromedp无头浏览器渲染作为高级可选模式，用户遇到极端排版精度需求时可手动切换，该模式不作为默认方案，不内置到安装包核心程序中
3.  导出强制规则：
    - 严格遵循A4标准纸张尺寸，自动处理`break-inside: avoid`规则，避免标题与内容跨页、段落强制拆分
    - 导出PDF与编辑器预览区采用完全一致的渲染引擎与CSS样式，确保排版100%一致，无错位、断行、样式偏差
    - 支持用户自定义导出页码范围、是否隐藏分页线、导出DPI参数
4.  导出流程：用户点击右上角「导出PDF」按钮 → 弹出保存对话框，用户选择保存路径与文件名 → 系统执行导出，实时展示进度 → 导出完成后弹窗提示，支持一键打开文件所在目录

#### 3.4.2 简历版本管理功能
1.  版本快照生成：
    - 支持用户手动创建版本快照，可自定义版本标签（如「v1.0 字节跳动投递版」）、版本备注
    - 自动快照规则：用户每次成功导出PDF后，系统自动创建一个版本快照，避免用户误操作导致内容丢失
2.  版本管理能力：
    - 支持查看当前简历的全量历史版本列表，展示版本标签、创建时间、修改人、备注信息
    - 支持两个版本之间的Diff对比，可视化展示内容差异
    - 支持一键回滚到指定历史版本，回滚前自动为当前内容创建快照，避免回滚误操作
3.  存储规则：版本快照完整存储对应版本的结构化JSON、Markdown内容、模板信息、自定义CSS，确保回滚后完全还原当时的编辑状态

#### 3.4.3 数据备份与恢复功能
1.  手动备份：支持用户一键导出全量数据备份包，备份包包含所有简历数据、版本快照、AI对话历史、用户配置信息，备份包为加密压缩格式，仅可通过本软件解密导入
2.  自动备份：支持用户设置自动备份周期，可选「每日、每周、每月」，自动备份到用户指定的本地文件夹，可设置最大自动备份数量，超出后自动覆盖最早的备份
3.  数据恢复：支持用户选择本地备份包，一键导入恢复全量数据，恢复前自动为当前现有数据创建备份，避免数据覆盖丢失

#### 3.4.4 本地数据基础管理
1.  简历列表管理：软件启动后默认进入简历列表页，展示本地存储的所有简历，支持按修改时间排序、搜索、重命名、复制、删除操作
2.  回收站功能：删除的简历进入回收站，保留30天，30天后自动清理，支持用户手动恢复、永久删除
3.  配置管理：支持用户管理API配置、默认模板、编辑偏好、快捷键设置，所有配置均本地存储

## 4. 技术方案
### 4.1 技术栈选型
严格遵循轻量化、高性能、开发者友好的选型原则，核心选型如下：
- 跨平台框架：Wails v2（Golang + WebView2/WebKit），替代Electron实现小包体积、低内存占用，MVP阶段安装包体积控制在50MB以内
- 后端语言：Golang，负责文件IO、SQLite数据库事务、AI API接口调用、系统级PDF打印接口、核心业务逻辑处理
- 前端技术栈：Vue 3 + TypeScript，负责UI构建、Monaco Editor状态管理、Markdown渲染、用户交互处理
- 编辑器内核：Monaco Editor（VS Code内核），确保编辑体验贴合开发者习惯
- 渲染引擎：Markdown-it，统一负责Markdown内容的HTML渲染，确保预览与导出渲染一致性
- 本地数据库：SQLite 3，负责全量本地数据存储
- PDF导出核心方案：Wails原生系统打印接口，Chromedp作为可选高级方案

### 4.2 架构设计与分层规范
严格遵循高内聚、低耦合的设计原则，采用分层架构设计，各层职责清晰、边界明确，可扩展性强。
#### 4.2.1 架构分层与职责
1.  **Bridge层（Wails绑定层）**
    - 职责：仅负责前端与后端的通信路由，不包含任何业务逻辑
    - 规范：所有前后端通信必须通过该层中转，定义统一的请求/响应格式，禁止前端直接调用底层能力
2.  **Domain业务层（核心业务逻辑层）**
    - 职责：封装所有核心业务逻辑，是系统的核心层
    - 核心模块：
      - `ResumeManager`：负责简历创建、编辑、模板填充、版本管理、数据同步全流程业务逻辑
      - `AIAgentService`：负责Prompt模板构建、API接口通信、流式响应处理、上下文管理、输出格式校验
      - `TemplateRender`：负责模板管理、样式渲染、JSON-MD数据转换
      - `ExportService`：负责PDF导出、数据备份/恢复核心逻辑
3.  **Infrastructure基础设施层**
    - 职责：负责底层基础设施能力的封装，为业务层提供无状态的基础能力
    - 核心模块：
      - `SQLiteStore`：负责数据库读写操作，封装统一的增删改查接口
      - `PDFRenderer`：封装原生系统打印与Chromedp两种PDF渲染能力，提供统一的调用接口
      - `HTTPClient`：统一处理AI API请求，封装重试、超时、错误处理逻辑
      - `EncryptService`：负责API Key等敏感数据的加密解密，统一实现AES-256-GCM加密算法
4.  **Provider适配层**
    - 职责：定义统一的接口规范，针对不同的实现提供适配器，实现底层能力与业务逻辑解耦
    - 规范：AI接口、PDF渲染能力均通过该层适配，后续新增模型、新增渲染方案仅需新增适配器，无需修改业务层代码

#### 4.2.2 核心设计原则
1.  高内聚：每个模块仅负责单一职责，相关逻辑完全内聚在对应模块中，无分散的业务逻辑
2.  低耦合：模块间通过统一的接口通信，无直接依赖，修改单个模块不影响其他模块的正常运行
3.  数据隔离：前端UI状态与持久化数据完全分离，所有持久化操作必须通过Go端的Repository层中转，禁止前端直接操作数据库
4.  样式驱动：所有渲染样式基于CSS变量实现，预留主题扩展能力，用户可通过配置调整核心样式参数，无需修改代码

### 4.3 核心技术规范（强制执行）
#### 4.3.1 前后端通信接口规范
1.  统一请求格式：所有接口请求采用统一的JSON格式，包含`request_id`、`module`、`action`、`data`四个核心字段
2.  统一响应格式：所有接口响应采用统一的JSON格式，包含`request_id`、`code`、`message`、`data`四个核心字段，定义统一的错误码体系
3.  流式传输规范：AI流式响应采用SSE（Server-Sent Events）协议，定义统一的消息格式，包含消息类型、内容、结束标识
4.  超时与重试规范：定义统一的接口超时时间，非幂等接口禁止自动重试，幂等接口失败后自动重试1次，重试失败后抛出异常

#### 4.3.2 Markdown渲染语法兼容标准
1.  核心兼容标准：严格遵循GitHub Flavored Markdown (GFM) 标准，确保语法兼容性
2.  简历专属扩展：仅支持表格、任务列表、脚注三个扩展语法，禁止其他非标准扩展语法，避免渲染异常
3.  渲染统一规则：预览区与PDF导出必须使用同一套Markdown渲染配置、同一套CSS样式表，确保渲染效果100%一致

#### 4.3.3 敏感数据加密存储规范
1.  加密算法：用户API Key等敏感数据必须采用AES-256-GCM算法加密存储，禁止明文存储
2.  密钥生成规则：加密密钥基于设备唯一标识+用户自定义盐值生成，仅当前设备可解密，即使数据库文件泄露，也无法解密敏感数据
3.  内存安全规范：敏感数据在内存中使用后立即清理，禁止长期留存，禁止通过日志输出敏感数据

## 7. 全场景异常处理与兜底策略
本章节为全系统强制执行的兜底规则，所有功能模块必须严格遵循，杜绝用户数据丢失、体验崩盘、软件崩溃等问题。

### 7.1 核心兜底原则
1.  数据永不丢失原则：任何异常场景下，必须完整保留用户的原始输入内容，绝不因异常、重试、校验失败导致用户内容丢失
2.  异常可感知原则：所有异常场景必须向用户明确提示错误原因与可执行的解决方案，禁止无提示的静默失败、白屏、崩溃
3.  操作可回滚原则：所有核心操作（回滚、删除、批量修改）执行前，必须自动创建数据快照，支持用户回滚到操作前状态
4.  降级可用原则：核心功能出现异常时，必须提供降级可用方案，确保用户可继续完成核心操作，如AI接口异常时，用户仍可正常编辑、保存、导出简历

### 7.2 分模块异常兜底规则
1.  **编辑器模块**：编辑器初始化失败自动降级为纯文本编辑模式；内容自动保存失败弹窗提示并定时重试；预览渲染异常自动重置为默认样式
2.  **AI智能体模块**：严格执行3.2.4章节定义的四大场景兜底策略，确保无内容丢失
3.  **渲染与模板模块**：模板加载失败自动降级为默认极简模板；自定义CSS渲染异常自动禁用自定义样式；JSON-MD同步失败保留原始数据，绝不自动覆盖用户内容
4.  **数据与导出模块**：PDF导出失败提供渲染模式切换方案；数据库读写失败自动创建临时备份；备份/恢复失败不修改用户原有数据；版本回滚失败自动恢复原有内容
5.  **全局异常处理**：启动时数据库损坏自动备份并尝试修复；全局未捕获异常自动捕获并记录日志，保留用户操作内容，禁止直接崩溃退出；配置加载失败自动恢复默认配置
