# 1. 文档信息
- 项目名称：Darvin-Resume
- 版本：v1.0
- 目标平台：Windows, macOS, Linux
- 核心定位：一款基于 Markdown、隐私优先、由 AI 深度赋能的本地化简历制作与管理工具。

# 2. 项目概述
## 2.1 产品目标
- 解决开发者在制作简历时遇到的排版难、内容润色难、多版本管理乱的问题。
- 通过本地部署确保隐私，利用 AI 接口提升内容质量。
- 提供接近 VS Code 的专业交互体验。

## 2.2 核心价值
- 本地化：数据存储在本地 SQLite，API Key 用户自备。
- AI 协同：不仅是编辑器，更是简历专家（基于 Claude API）。
- 所见即所得：实时渲染 Markdown 为极简风格 PDF。

# 3. 功能需求
## 3.1 核心编辑器模块
- 新建简历
  - 支持创建新简历，并选择“AI 引导写简历”或“空白页面”。
  - AI 引导写简历包含步骤：简历信息、校园经历、专业技能、实习经历、项目经历、自我评价。
  - 用户根据提示输入信息，AI 负责润色并生成符合模板的 Markdown 文档。
  - 选择空白页面后，可进入简历编写界面，自行填写并可随时调用 AI 进行写作或润色。
- Markdown 编辑
  - 采用 Monaco Editor（VS Code 内核）。
  - 支持语法高亮与自动补全。
- 实时预览
  - 双栏布局，左侧编辑 Markdown，右侧实时渲染 HTML 预览。
- 交互增强
  - 选区操作：框选文字后弹出浮动工具栏（类似 Medium/VS Code），包含“AI 润色”、“翻译”、“缩写”等快捷动作。
  - 行级交互：点击行首符号可快速移动块或触发 AI 询问。

## 3.2 AI 智能体模块 (AI Agent)
- 接口适配
  - 首选集成 Claude Messages API。
  - 支持自定义 BaseURL 以适配中转站。
- 数据流式传输（Streaming）
  - 提升交互丝滑感。
- 智能填充与原子化
  - AI 填充基于 JSON Schema 结构，而不是整段死文本，前端根据 Schema 渲染成 Markdown。
  - 支持“跳过”和“局部重生成”，用户可修改单个字段后由 AI 精准重写该字段，而不重写整份简历。
- 上下文感知
  - 当用户框选文本弹出 AI 弹窗时，发送给 Claude 的 Prompt 应包含当前选中文本、简历整体职位目标（如“Golang 开发”）、当前 Markdown 模板样式说明。
  - 增加“全文参考”开关，决定 AI 润色时是否考虑整篇简历的语气一致性。
- 交互模式
  - 侧边栏聊天（Chat Window）：支持多轮对话，可引用选中的文本作为上下文。
  - 引导式填写：AI 像面试官一样提问（“你的核心技术栈是什么？”），根据用户回答自动生成 Markdown 代码块。
  - 一键解析：支持将旧的文本内容或结构化信息一键转化为符合当前模板的 Markdown。

## 3.3 模板与渲染系统
- 样式驱动
  - 默认内置“冷熊简历”极简风格。
  - 支持多套预设模板（如：大厂风、学术风、双栏简约）。
- 自定义接口
  - 用户可自定义 CSS 文件覆盖默认样式。
  - 通过占位符（如 `{{name}}`）支持从结构化 JSON 填充模板。
- 数据一致性策略 (JSON-MD Bridge)
  - 底层数据模型：采用结构化 JSON 存储核心履历（Education, Projects, Skills）。
  - 视图层转换：由 Go 后端负责将 JSON 数据套入 Markdown 模板进行渲染。
  - 双向反馈：AI 润色优先操作 JSON 数据，确保简历逻辑结构不因 AI 的随机性而乱序；MD 编辑器仅作为用户微调样式的“展示壳”。

## 3.4 数据与导出
- 本地数据库：使用 SQLite 存储简历版本、AI 对话历史、用户配置。
- PDF 导出：支持一键调用 Headless Browser 将当前渲染页面转换为 A4 标准 PDF，确保排版不偏移。
- PDF 分页控制：渲染引擎自动处理 `break-inside: avoid`，避免断行导致标题与内容跨页。
- 分页提示：实时预览区增加分页线模拟，让用户在编辑时感知 A4 纸张边界，避免导出后排版错乱。

# 4. 技术方案 (High-Level Architecture)
## 4.1 技术栈选型
- 跨平台框架：Wails（Golang + WebView2/WebKit）。相比 Electron，包体积更小，内存占用更低。
- 后端（Go）：负责文件 IO、SQLite 事务、Claude API 调用、系统级 PDF 打印接口。
- 前端（Vue 3 + TS）：负责 UI 构建、Monaco 编辑器状态、Markdown-it 渲染。
- 数据库：SQLite 3。

## 4.2 软件工程设计原则
- 高内聚（High Cohesion）
  - AI-Service 专门负责 Prompt 构建与接口通信。
  - Export-Service 专门处理 HTML 转 PDF 逻辑。
- 低耦合（Low Coupling）
  - Provider 模式：定义统一的 AI 接口规范，未来更换 GPT-5 或本地 Ollama 仅需新增适配器。
  - 数据隔离：UI 状态与持久化数据分离，通过 Go 端的 Repository 层进行中转。
- 架构层次
  - Bridge 层（Wails 绑定）：仅负责前端与后端的通信路由，不写业务逻辑。
  - Domain 业务层：
    - `ResumeManager`：处理简历创建、编辑、模板填充、版本管理等逻辑。
    - `AIAgent`：处理 Prompt 模板、流式响应和上下文管理。
  - Infrastructure 基础设施层：
    - `SQLiteStore`：数据库读写。
    - `PDFRenderer`：调用 Chromedp 进行无头浏览器打印。
    - `HTTPClient`：统一处理 Claude API 请求。
- 样式驱动与主题变量
  - 支持 CSS 变量，预留 Theme Variables，例如 `--primary-color`、`--font-size-base`、`--line-height`。
  - 用户可通过配置或滑块调整主色调、字号与行高，而不必编写复杂 CSS。

# 5. UI/UX 设计要求
## 5.1 视觉风格
- 极简主义：克制的色彩使用，聚焦文字内容。
- 黑暗模式：支持根据系统自动切换深色/浅色皮肤。

## 5.2 交互逻辑
- 启动：显示简历列表页（本地 SQLite 数据）。
- 新建：支持“AI 引导写简历”和“空白页面”两种新建方式。
- AI 引导写简历：按步骤依次填写简历信息、校园经历、专业技能、实习经历、项目经历、自我评价，AI 实时润色并填充符合模板的 Markdown 内容。
- 空白页面：进入简历编辑界面，自行填写或随时调用 AI 进行写作和润色。
- 编辑：进入编辑器，左侧源码，右侧预览。
- AI 操作：框选一段经历 -> 点击浮窗“润色” -> 右侧 Chat 弹窗显示建议 -> 点击“接受” -> 源码自动替换。
- 编辑增强：利用 Monaco Editor 的 DeltaDecorations，以 Diff 视图（左红右绿）显示 AI 润色建议，用户点击“接受”才应用修改。
- 空状态设计：当没有简历时，首页直接展示“冷熊简历”样式的 Demo 预览，激发用户创作欲望。
- 页面边界：实时预览区显示 A4 分页线，帮助用户判断导出时的排版位置。
- 导出：点击右上角导出 -> 弹出保存对话框 -> 获得本地 PDF。

# 6. 数据存储结构 (SQLite Schema 预览)
- `resumes`：存储简历元数据。建议字段：
  - `id` UUID 主键
  - `title` String 简历名称
  - `content_md` Text Markdown 源码
  - `content_json` Text AI 引导填写的结构化数据，方便二次修改
  - `template_id` String 使用的模板 ID
  - `custom_css` Text 该简历特有的样式覆盖
  - `last_modified` DateTime 最后修改时间
  - `version_tag` String 版本标签（如：v1.1 投递字节版）
- `configs`：存储加密后的 API Key、BaseURL、用户基本偏好。
- `ai_history`：存储与当前简历相关的对话上下文。
